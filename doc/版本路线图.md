## 1. 当前基线：v0.1.0

**发布日期**：2026-06-01
**commit**：`cbe52da`（feat: pi-marketplace v0.1.0 — search, audit, install pi packages with security review）

**已有能力**：
- 4 个 LLM tool：`marketplace_search` / `marketplace_detail` / `marketplace_audit` / `marketplace_install`
- `extensions/security.ts` 两层审计（metadata + source scan）
- 14 个测试（unit + integration）
- 无 UI 入口、无 i18n、无 scope 选择、无 fallback 策略

## 2. 版本演进计划

### v0.2.0 — 工具质量（最小增量）

| 字段 | 值 |
|---|---|
| 主题 | 让 LLM 知道何时调用我们的工具 |
| 工作量 | 1-2 天 |
| 风险 | **低**（纯增量） |
| 依赖 | 无 |
| 关联 | 借鉴路线图 § 2.1 #1、#2 |

**包含**：
- **#1** `promptSnippet` + `promptGuidelines` 补齐（4 个 tool 都加）
- **#2** `security.ts` 三项同步：`maxBuffer: 32MB` / `AbortSignal` / HTTP URL 白名单 `?!registry.npmjs.org|api.npmjs.com`

**不做**：
- 任何 UI 改动
- 任何数据模型改动
- 任何目录结构改动

**验证标准**：
- 4 个 tool 都有 `promptSnippet` / `promptGuidelines` 字段
- `security.ts` 测试覆盖 `maxBuffer`、`AbortSignal` 取消、URL 白名单过滤
- 现有 14 个测试仍通过
- `npm run typecheck` 通过

---

### v0.3.0 — 体验优化

| 字段 | 值 |
|---|---|
| 主题 | 安装过程可观察、目录结构清晰 |
| 工作量 | 2-3 天 |
| 风险 | **低-中**（目录重组要改 import 路径） |
| 依赖 | 无（v0.2.0 之后做） |
| 关联 | 借鉴路线图 § 2.1 #3、#4 |

**包含**：
- **#3** 安装进度：`marketplace_install` 调 `setStatus` 显示 spinner
- **#4** 目录重组：`tools/` 下 4 个并列文件 → 合并为 `tools/index.ts`（~200 行）或按"搜索-详情"/"审计-安装"分 2 文件

**不做**：
- i18n（推到 v0.4.0）
- 任何数据模型改动

**验证标准**：
- `marketplace_install` 执行时 `setStatus` 调用可观察
- `tools/` 目录结构清晰
- 现有 14 个测试仍通过

### v0.4.0 — i18n 地基

| 字段 | 值 |
|---|---|
| 主题 | UI 文案可翻译，为 v0.5.0 数据模型和 slash command 铺路 |
| 工作量 | 1 周 |
| 风险 | **中-高**（所有 UI 文案需扫一遍） |
| 依赖 | v0.3.0（先把目录理清再上 i18n） |
| 关联 | 借鉴路线图 § 2.1 #5 |

**包含**：
- **#5** i18n 基础设施：
  - 新建 `extensions/i18n.ts`，暴露 `t(key, locale?)` 函数
  - 硬编码 zh-CN + en 两套
  - 当前 4 个 tool 的所有 UI 文案走 `t('xxx')`
  - demo 切换方式：通过环境变量或临时 `setLocale()`

**不做**：
- 5 语言 i18n
- 翻译缓存
- 完整多语言覆盖（只覆盖当前 UI 元素）

**验证标准**：
- 存在 `extensions/i18n.ts`
- 当前 4 个 tool 内部 `ctx.ui.notify` / `ctx.ui.confirm` 等文案全部走 `t()`
- 切换 locale 能看到不同文案（至少在测试中验证）

---

### v0.5.0 — 中期改进（数据 + 入口）

| 字段 | 值 |
|---|---|
| 主题 | 数据更丰富，用户能离 AI 走流程 |
| 工作量 | 2-3 周 |
| 风险 | **高**（数据模型扩展影响所有 tool） |
| 依赖 | v0.4.0（i18n 地基先打） |
| 关联 | 借鉴路线图 § 2.2 #6-#11 |

**包含**：
- **#6** `PackageInfo` 模型扩展：`installed` / `installedVersion` / `scope` / `sourceType` / `pinned` / `hasUpdate` / `skipReason`
- **#7** slash command `/marketplace`（handler 调简单 `select` 菜单）
- **#8** scope 选择（Global / Project）
- **#9** fallback 策略（uninstall 失败时直接改 settings）
- **#10** 测试分层（unit 纯函数 + integration IO）
- **#11** 完整 `PackageInfo` 字段数据流（`formatPackageDetail` / `formatAuditReport` 读新模型）

**不做**：
- overlay 完整面板（推到 v0.6.0）
- 完整 i18n
- 自定义 TUI 组件

**验证标准**：
- `PackageInfo` 包含上述 7 个扩展字段
- `/marketplace` 能开菜单
- 安装时可选 Global / Project
- 卸载失败时 fallback 改 settings
- 测试能区分 unit / integration 两层

---

### v0.6.0+ — 长期（overlay 完整版）

| 字段 | 值 |
|---|---|
| 主题 | TUI overlay 完整版 + 完整 i18n |
| 工作量 | 按需拆分多个 minor |
| 风险 | 高 |

**可能拆分**：

- **v0.7.0** 自定义 `PackageList` 组件 + filter chips / 快捷键
- **v0.8.0** 完整 i18n（5 语言 + 翻译缓存 + override）
- **v0.9.0** README 渲染 / AI 语义搜索（可选）

**明确不做**（借鉴路线图 § 3）：
- ❌ 多 tab（Installed / Browse / Updates）
- ❌ AI 语义搜索（推到 v1.0+）
- ❌ pi.dev enrichment

## 4. CHANGELOG 策略

**v0.2.0 起**开始维护 `CHANGELOG.md`，每个版本一个 section，关联借鉴路线图的项：

```markdown
# Changelog

## [v0.2.0] - TBD

### 改进
- 4 个 LLM tool 补齐 `promptSnippet` / `promptGuidelines` 字段
- `security.ts` 同步：`maxBuffer` 32MB、`AbortSignal` 支持、HTTP URL 白名单

### 关联
- 借鉴路线图 § 2.1 #1、#2
- 版本路线图 v0.2.0
```

**v0.1.0 之前**不补历史 CHANGELOG（项目太小，补历史没意义）。
