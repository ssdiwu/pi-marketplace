## 1. 当前基线：v0.1.0

**发布日期**：2026-06-01
**commit**：`cbe52da`（feat: pi-marketplace v0.1.0 — search, audit, install pi packages with security review）

**已有能力**：
- 4 个 LLM tool：`marketplace_search` / `marketplace_detail` / `marketplace_audit` / `marketplace_install`
- 每个 tool 都带 `promptSnippet` + `promptGuidelines` 字段（LLM 协作精准基线）
- `extensions/security.ts` 两层审计（metadata + source scan）
- 14 个测试（unit + integration）
- 无 UI 折叠/展开、无 i18n、无 scope 选择、无 fallback 策略

## 2. 版本演进计划

> **设计原则**：每个 minor / patch 版本讲一个**完整故事**，不是每个 feature 一个版本号。**编号沿用借鉴路线图 § 2.1 / § 2.2 / § 2.3 / § 2.1.1 编号**，方便读者一眼对应"学什么"。

### v0.1.1 — 渲染层增强（独立 patch，2026-06-07 即将发布）

| 字段 | 值 |
|---|---|
| 主题 | Tool 输出支持折叠/展开 |
| 工作量 | 0.5-1 天 |
| 风险 | **低**（纯 UI 增强，`execute` 返回值不变） |
| 依赖 | 无 |
| 关联 | 借鉴路线图 § 2.1.1 |
| 实现 commit | `0fcee91` |

**包含**：
- 4 个 tool（search / detail / audit / install）加 `renderResult` 字段
- 新建共享 `extensions/render.ts`（~100 行）
  - `renderCollapsibleMarkdown` 通用函数
  - `formatRiskBadge` helper
- 加 peer/dev dep `@earendil-works/pi-tui@^0.78.1`

**不做**：
- 任何数据模型改动
- 任何 tool `execute` 行为改动
- 任何 i18n / 翻译改动

**验证标准**：
- `/reload` 后 4 个 tool 都显示折叠态（1 行摘要 + `(ctrl+o to expand)`）
- 按 `ctrl+o` 展开看到完整 markdown
- 4 个 tool 摘要都对：
  - search → `📦 Found N pi package(s)`
  - detail → `📦 {name} v{version}`
  - audit → `🔒 {name}: {RISK_BADGE} (N finding(s))`
  - install → `📥 {result} {name} (Audit: {RISK_BADGE})`
- 现有 5/5 测试通过
- `npm run typecheck` 通过
- `npm pack --dry-run` tarball 不含 `doc/`

---

### v0.2.0 — 短期迭代（借鉴路线图 § 2.1 #2-#5，4 项）

| 字段 | 值 |
|---|---|
| 主题 | 补齐工具质量 + 体验优化 + i18n 地基 |
| 工作量 | 1-1.5 周 |
| 风险 | **中**（4 项叠加，但单项都简单） |
| 依赖 | v0.1.1（无强依赖，但建议等 v0.1.1 落地后开始） |

**包含**（沿用借鉴路线图 § 2.1 编号，#1 已在 v0.1.0 完成）：
- **#2** `security.ts` 三项同步：`maxBuffer: 32MB` / `AbortSignal` / HTTP URL 白名单 `?!registry.npmjs.org|api.npmjs.com`
- **#3** 安装进度：`marketplace_install` 调 `setStatus` 显示 spinner
- **#4** 目录重组：`tools/` 下 4 个并列文件 → 合并为 `tools/index.ts`（~200 行）或按"搜索-详情"/"审计-安装"分 2 文件
- **#5** i18n 基础设施：新建 `extensions/i18n.ts`，暴露 `t(key, locale?)` 函数，硬编码 zh-CN + en 两套

**不做**：
- 任何数据模型改动
- 任何 slash command
- 任何 overlay 面板
- 完整 5 语言 i18n
- 翻译缓存

**验证标准**：
- `security.ts` 测试覆盖 `maxBuffer`、`AbortSignal` 取消、URL 白名单过滤
- `marketplace_install` 执行时 `setStatus` 可观察
- `tools/` 目录结构清晰
- 存在 `extensions/i18n.ts`，4 个 tool 内部 UI 文案走 `t()`
- 现有 5/5 测试仍通过
- `npm run typecheck` 通过

---

### v0.3.0 — 中期改进（借鉴路线图 § 2.2 #6-#11，6 项）

| 字段 | 值 |
|---|---|
| 主题 | 数据更丰富，用户能离 AI 走流程 |
| 工作量 | 2-3 周 |
| 风险 | **高**（数据模型扩展影响所有 tool） |
| 依赖 | v0.2.0（i18n 地基先打） |

**包含**（沿用借鉴路线图 § 2.2 编号）：
- **#6** `PackageInfo` 模型扩展：`installed` / `installedVersion` / `scope` / `sourceType` / `pinned` / `hasUpdate` / `skipReason`
- **#7** slash command `/marketplace`（handler 调简单 `select` 菜单）
- **#8** scope 选择（Global / Project）
- **#9** fallback 策略（uninstall 失败时直接改 settings）
- **#10** 测试分层（unit 纯函数 + integration IO）
- **#11** 完整 `PackageInfo` 字段数据流（`formatPackageDetail` / `formatAuditReport` 读新模型）

**不做**：
- overlay 完整面板（推到 v0.4.0+）
- 完整 i18n
- 自定义 TUI 组件

**验证标准**：
- `PackageInfo` 包含上述 7 个扩展字段
- `/marketplace` 能开菜单
- 安装时可选 Global / Project
- 卸载失败时 fallback 改 settings
- 测试能区分 unit / integration 两层

---

| 字段 | 值 |
|---|---|

| 工作量 | 按需拆分多个 minor |
| 风险 | 高 |
| 依赖 | v0.3.0 |

**可能拆分**（沿用借鉴路线图 § 2.3 编号）：
- **v0.4.0** = #12 TUI overlay 面板基础（`ctx.ui.custom` + `panelLoop` 重入）+ #13 `panelLoop` 重入
- **v0.5.0** = #14 自定义 `PackageList` 组件 + #15 filter chips / 快捷键
- **v0.6.0** = #16 完整 i18n（5 语言 + 翻译缓存 + override）
- **v0.7.0** = #17 README 渲染 / #18 AI 语义搜索（可选）

**明确不做**（借鉴路线图 § 3）：
- ❌ 多 tab（Installed / Browse / Updates）
- ❌ AI 语义搜索（推到 v1.0+）
- ❌ pi.dev enrichment
