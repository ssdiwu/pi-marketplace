## 1. 当前基线：v0.1.0

**发布日期**：2026-06-01
**commit**：`cbe52da`（feat: pi-marketplace v0.1.0 — search, audit, install pi packages with security review）

**已有能力**：

## 2. 版本演进计划

> **设计原则**：每个 minor 版本讲一个**完整故事**，不是每个 feature 一个版本号。短期 5 项放一起 = v0.2.0，中期 6 项放一起 = v0.3.0，以此类推。如果某个版本内某项需要单独发 patch，可以用 v0.2.1 / v0.2.2。

### v0.2.0 — 短期迭代（借鉴路线图 § 2.1 全部 5 项）

| 字段 | 值 |
|---|---|
| 主题 | 补齐工具质量 + 体验优化 + i18n 地基 |
| 工作量 | 1-2 周 |
| 风险 | **中**（5 项叠加，但单项都简单） |
| 依赖 | 无（v0.1.0 之后即可开始） |

**包含**（借鉴路线图 § 2.1）：
- **#1** `promptSnippet` + `promptGuidelines` 补齐（4 个 tool 都加）
- **#2** `security.ts` 三项同步：`maxBuffer: 32MB` / `AbortSignal` / HTTP URL 白名单 `?!registry.npmjs.org|api.npmjs.com`
- **#3** 安装进度：`marketplace_install` 调 `setStatus` 显示 spinner
- **#4** 目录重组：`tools/` 下 4 个并列文件 → 合并为 `tools/index.ts`（~200 行）或按"搜索-详情"/"审计-安装"分 2 文件
- **#5** i18n 基础设施：新建 `extensions/i18n.ts`，暴露 `t(key, locale?)` 函数，硬编码 zh-CN + en 两套

**不做**：
- 任何数据模型改动
- 任何 slash command
- 任何 overlay 面板
- 完整 5 语言 i18n
- 翻译缓存

**验证标准**：
- 4 个 tool 都有 `promptSnippet` / `promptGuidelines` 字段
- `security.ts` 测试覆盖 `maxBuffer`、`AbortSignal` 取消、URL 白名单过滤
- `marketplace_install` 执行时 `setStatus` 可观察
- `tools/` 目录结构清晰
- 存在 `extensions/i18n.ts`，4 个 tool 内部 UI 文案走 `t()`
- 现有 14 个测试仍通过
- `npm run typecheck` 通过

---

### v0.3.0 — 中期改进（借鉴路线图 § 2.2 全部 6 项）

| 字段 | 值 |
|---|---|
| 主题 | 数据更丰富，用户能离 AI 走流程 |
| 工作量 | 2-3 周 |
| 风险 | **高**（数据模型扩展影响所有 tool） |
| 依赖 | v0.2.0（i18n 地基先打） |

**包含**（借鉴路线图 § 2.2）：
- **#6** `PackageInfo` 模型扩展：`installed` / `installedVersion` / `scope` / `sourceType` / `pinned` / `hasUpdate` / `skipReason`
- **#7** slash command `/marketplace`（handler 调简单 `select` 菜单）
- **#8** scope 选择（Global / Project）
- **#9** fallback 策略（uninstall 失败时直接改 settings）
- **#10** 测试分层（unit 纯函数 + integration IO）
- **#11** 完整 `PackageInfo` 字段数据流（`formatPackageDetail` / `formatAuditReport` 读新模型）

**不做**：
- overlay 完整面板（推到 v0.4.0）
- 完整 i18n
- 自定义 TUI 组件

**验证标准**：
- `PackageInfo` 包含上述 7 个扩展字段
- `/marketplace` 能开菜单
- 安装时可选 Global / Project
- 卸载失败时 fallback 改 settings
- 测试能区分 unit / integration 两层

---

### v0.4.0+ — 长期（借鉴路线图 § 2.3 全部 7 项，按需拆分）

| 字段 | 值 |
|---|---|
| 主题 | TUI overlay 完整版 + 完整 i18n + 锦上添花 |
| 工作量 | 按需拆分多个 minor |
| 风险 | 高 |
| 依赖 | v0.3.0 |

**可能拆分**：
- **v0.4.0** TUI overlay 面板基础（`ctx.ui.custom` + `panelLoop` 重入）
- **v0.5.0** 自定义 `PackageList` 组件 + filter chips / 快捷键
- **v0.6.0** 完整 i18n（5 语言 + 翻译缓存 + override）
- **v0.7.0** README 渲染 / AI 语义搜索（可选）

**明确不做**（借鉴路线图 § 3）：
- ❌ 多 tab（Installed / Browse / Updates）
- ❌ AI 语义搜索（推到 v1.0+）
- ❌ pi.dev enrichment

## 4. CHANGELOG 策略

**v0.2.0 起**开始维护 `CHANGELOG.md`，每个版本一个 section，关联借鉴路线图的项：

```markdown

## [v0.2.0] - TBD

### 改进
- 4 个 LLM tool 补齐 `promptSnippet` / `promptGuidelines` 字段
- `security.ts` 同步：`maxBuffer` 32MB、`AbortSignal` 支持、HTTP URL 白名单
- `marketplace_install` 安装时显示进度（`setStatus` spinner）
- `tools/` 目录重组：4 个并列文件 → 合并/分组
- 新建 `extensions/i18n.ts`，4 个 tool 内部 UI 文案走 `t()`，硬编码 zh-CN + en

### 关联
- 借鉴路线图 § 2.1 #1、#2、#3、#4、#5
- 版本路线图 v0.2.0
```
